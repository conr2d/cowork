**v0.1 single goal:** on Windows, take a non-developer from "nothing" to "an AI coding agent installed inside WSL2, logged in, with a working terminal." **Setup only.**

## Setup flow (wizard, "all the way")

1. **Preflight** â€” run the checks below; render each blocker explicitly.

   | Check | Hard threshold | kind on fail | code | Detection |
   |---|---|---|---|---|
   | Windows build | â‰¥ 19041 (Win10 2004) | `Blocker` | `preflight.windows_build_unsupported` | `RtlGetVersion` / registry `CurrentBuildNumber`+`UBR` |
   | CPU arch | x64 or arm64 | `Blocker` | `preflight.arch_unsupported` | `GetNativeSystemInfo` |
   | CPU virtualization capability | VT-x/AMD-V present | `Blocker` | `preflight.virtualization_unsupported` | CPU feature query |
   | Firmware virtualization enabled | enabled | `NeedsUserAction` | `preflight.virtualization_disabled` | WMI `Win32_Processor.VirtualizationFirmwareEnabled` **cross-checked with `HypervisorPresent`** (avoid false-negative when Hyper-V already owns VT-x) |
   | Hypervisor conflict | VMP usable / no blocking 3rd-party VMM | `NeedsUserAction` | `preflight.hypervisor_conflict` | `HypervisorPresent` + known-conflict drivers |
   | Free disk (WSL target volume) | â‰¥ 16 GiB (rec. 32) | `NeedsUserAction` | `preflight.insufficient_disk` | `GetDiskFreeSpaceEx` |
   | Admin / elevation | UAC elevation obtainable | `Blocker` | `preflight.elevation_unavailable` | `CheckTokenMembership`(Administrators) |
   | **Enterprise policy (office-PC critical)** | WSL not policy-blocked; Store usable; CFA not blocking | `Blocker`/`NeedsUserAction` | `preflight.wsl_blocked_by_policy`, `preflight.store_disabled`, `preflight.controlled_folder_access` | Group Policy/MDM registry, Store state, Defender Controlled Folder Access |

   **Not gated:** Windows edition (WSL2 runs on Home via VirtualMachinePlatform). WSL app version (â‰¥2.4.4) checked at wsl-enable after `--update`. Network is a non-blocking ping; real failures surface later (`common.network_failed`, `common.proxy_tls_error`).
   **Elevation nuance:** `preflight.elevation_unavailable` (`Blocker`) is for a non-admin account with no obtainable admin; a user clicking "No" on UAC mid-flow is `wsl.elevation_denied` (`NeedsUserAction`, retryable).

2. **WSL enable/update** â€” `wsl --install --no-distribution` + `wsl --update` via UAC elevation; verify WSL app â‰¥ 2.4.4. **Reboot-resume:** before a required reboot, write `Cowork.exe --resume` to the Windows **`RunOnce`** registry key so the wizard relaunches automatically (a non-developer will not relaunch manually); persist wizard state; on resume, validate state (`host.resume_state_corrupt`). Note the `wsl --update` Catch-22: build 19041's inbox `wsl.exe` may not support modern flags before update â€” handle via `wsl.update_unsupported_inbox` (guide to Store/MSI update).

3. **Provision distro** â€” create the **Cowork-dedicated distro** (decision #2), never disturbing an existing `Ubuntu`. Cover: for the `--import` path, rootfs fetch (`distro.rootfs_fetch_failed`) + SHA256 verify (`distro.checksum_mismatch`) + import (`distro.import_failed`); for the `--name` path, `distro.install_failed`; an existing `Cowork` distro â†’ idempotent re-use or explicit re-provision (`distro.already_exists`); firstboot user creation (`distro.user_create_failed`); disk exhaustion (`distro.disk_full`). Then **inject the guest CLI**: the host copies the cross-compiled Linux `cowork` binary into the distro (via `\\wsl.localhost\Cowork\...` or piping through `wsl.exe`) before invoking `wsl.exe -d Cowork -- cowork ...` (`guest.cli_inject_failed`).

4. **Toolchain bootstrap** (`cowork` CLI, inside WSL) â€” `apt install` Linuxbrew prerequisites first (`build-essential`, `curl`, `git`, `file`, `procps` â†’ `toolchain.prereq_apt_failed`); install Linuxbrew (`/home/linuxbrew/.linuxbrew`) + mise; wire shellrc (`brew shellenv`, `mise activate`); if codex selected, pin **Node 24 LTS (exact patch)** via mise. Locales: `locale-gen` ja_JP/ko_KR/en_US.

5. **Agent install** (decision #9) â€” **claude** and **antigravity** via native installers (run with unattended flags to avoid TTY hangs â†’ `agent.installer_hang`); **codex** (`@openai/codex`) via `~/.default-npm-packages` on the pinned Node. Verify each binary post-install (`agent.binary_not_found`, `agent.integrity_check_failed`). Route creds to `~/.cowork/creds/` (decision #13).

6. **Auth** â€” open the embedded xterm.js terminal; run each agent's native login (first-run interactive OAuth). OAuth bounces to the **host browser via WSLInterop** (`$BROWSER`/`xdg-open`); if Interop/`$BROWSER` is unavailable, fall back to printing the URL / device-code for manual completion (`auth.browser_launch_failed`, `auth.oauth_callback_failed`, `auth.login_timeout`). Creds land under `~/.cowork/creds/` per #13.

7. **Done** â€” drop the user into the embedded terminal at **`~/cowork/`** (default workspace, created during bootstrap; the seed for v0.2 per-workspace isolation).

## Uninstall (v0.1, minimal)

A **Remove Cowork** action does a clean teardown: `wsl --unregister Cowork` (drops the dedicated distro and all its data), then clears host-side state (the `RunOnce` key if pending, persisted wizard state, `~/.cowork` host config). Clean precisely because the install is isolated to a dedicated distro â€” the user's other distros are untouched. Failure â†’ `distro.unregister_failed`.

## Architecture / crates

- **`src/`** â€” SvelteKit-static UI: wizard + embedded terminal. i18n via Paraglide-JS (ja+ko+en, en base).
- **`src-tauri/`** â€” Tauri v2 Rust, **Windows host driver** (OS-specific layer): preflight queries (incl. policy checks), `wsl.exe` driver, UAC elevation + `RunOnce` reboot-resume, ConPTYâ†”`wsl.exe` PTY bridge to xterm.js, structured progress events, **guest-CLI injection**. Builds `Cowork.exe`. Applies the 7 embedded-terminal tunings.
- **`cowork/`** â€” **host-agnostic guest CLI** (binary `cowork`) run inside WSL; performs bootstrap (apt prereqs/brew/mise/node/agent install), owns `~/.default-npm-packages` and the `~/.cowork/creds` routing, emits JSON-lines progress. Unit-testable (`cargo test -p cowork`). Future `coworkd` daemon (v0.5) lands beside it; reused as-is on any future host.
- **`templates/`** â€” minimal default workspace only in v0.1.
- (`distro-build/` is **not** in v0.1 â€” deferred with Stage B.)

## Error model â€” LOCKED (post-review)

The `cowork` CLI and `src-tauri` backend **never emit localized strings** â€” only a structured envelope; the frontend maps it to localized text.

- **Envelope:** `{ code, kind, stage, context, cause? }`.
  - `code` â€” stable dotted `<domain>.<reason>`; also the i18n key root (`errors.<code>.title`/`.body`). No numeric codes.
  - `kind` â€” `Blocker` (unbypassable here) Â· `NeedsUserAction` (fix outside app, retry) Â· `Transient` (retry/backoff) Â· `Internal` (bug â†’ report).
  - `stage` â€” `preflight | wsl-enable | provision | toolchain | agent-install | auth | done`.
  - `context` â€” camelCase key-values for ICU interpolation. `cause` â€” optional English diagnostic for logs/support, never shown localized, and **redacted** (strip usernames, home paths, tokens, URLs, OAuth artifacts) before any support export.
- **Single source of truth:** one checked-in `errors.json` (`code â†’ {kind, stage, contextKeys}`); `build.rs` codegens the Rust enum; the TS frontend imports the same file. Parity across Rust â†” TS â†” 3 locales is guaranteed by construction.
- **Network rule:** transient connectivity failures collapse to `common.network_failed` (Transient, keyed by `stage`); endpoint-specific failures (changed/404 installer URL, TLS interception) keep their own code because remediation differs.

## Operational diagnostics (known fragile paths)

The external-installer + auth steps are the top v0.1 failure variables and are explicitly tracked as **fragile/external** in a (redacted) support log bundle: agent installer URLs (`claude.ai/install.sh`, `antigravity.google/cli/install.sh`, npm registry), TLS/proxy interception, the OAuth flow, and each agent's actual credential path. These are **re-validated whenever an agent self-updates** (URLs/paths can change without notice â€” see WP7 pre-gate). The bundle redacts usernames / home paths / tokens / URLs / OAuth artifacts per the `cause` rules.

## Quality & conformance (applies to every WP)

Functional "Done when" is necessary but **not sufficient** â€” it guards against drift from the plan. A unit/WP is complete only when **all four** hold:

1. **Functional DoD** â€” the per-WP checks below.
2. **Automated conformance passes** (CI/lint/test â€” the *unit-seal gate*): host/guest separation (`cowork` has no Windows deps), no backend-emitted localized strings (codes only), `errors.json` single-source + 3-locale parity, envelope-only guestâ†’host messaging. A unit failing these **cannot be sealed/committed**. (Full rule list in [`../AGENTS.md`](../AGENTS.md) â†’ Quality & conformance.)
3. **Separate-context review sign-off** â€” architecture-conformance + code review by a *different* agent, with evidence; never self-approved. Risk-weighted: heaviest on WP4â€“WP8, light on WP0â€“WP2.
4. **Deviation recorded** â€” commit trailer for small departures; `docs/adr/NNNN-*.md` for architectural ones (with temporary/permanent + v0.2â€“v0.5 impact).

**Incremental integration:** WP4â€“WP8 each run their e2e slice on the Hyper-V checkpoint VM **as they finish** â€” integration is continuous, not deferred to WP10.
