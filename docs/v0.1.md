# Cowork v0.1 — Implementation Spec

> The v0.1 implementation spec. Strategy and the cross-version roadmap live in [`plan.md`](./plan.md); cross-tool working rules in [`../AGENTS.md`](../AGENTS.md).

> **⚠️ Historical. This is what v0.1 was planned to be, not what it became — read the source before following it.** Two of the decisions below were reversed *during* implementation and are corrected in place: **codex installs from its own native installer, not npm** (it went standalone, so no Node and no `~/.default-npm-packages` exist anywhere in the guest — `f6159b2`), and **credentials stay at each agent's default path**, never routed to `~/.cowork/creds/` (central routing fought the agents' own OAuth refresh — `eab7522`). The auth section is likewise obsolete: agents drive their own sign-in and Cowork runs no login command (`816317a`).


**v0.1 single goal:** on Windows, take a non-developer from "nothing" to "an AI coding agent installed inside WSL2, logged in, with a working terminal." **Setup only.**

## Setup flow (wizard, "all the way")

1. **Preflight** — run the checks below; render each blocker explicitly.

   | Check | Hard threshold | kind on fail | code | Detection |
   |---|---|---|---|---|
   | Windows build | ≥ 19041 (Win10 2004) | `Blocker` | `preflight.windows_build_unsupported` | `RtlGetVersion` / registry `CurrentBuildNumber`+`UBR` |
   | CPU arch | x64 or arm64 | `Blocker` | `preflight.arch_unsupported` | `GetNativeSystemInfo` |
   | CPU virtualization capability | VT-x/AMD-V present | `Blocker` | `preflight.virtualization_unsupported` | CPU feature query |
   | Firmware virtualization enabled | enabled | `NeedsUserAction` | `preflight.virtualization_disabled` | WMI `Win32_Processor.VirtualizationFirmwareEnabled` **cross-checked with `HypervisorPresent`** (avoid false-negative when Hyper-V already owns VT-x) |
   | Hypervisor conflict | VMP usable / no blocking 3rd-party VMM | `NeedsUserAction` | `preflight.hypervisor_conflict` | `HypervisorPresent` + known-conflict drivers |
   | Free disk (WSL target volume) | ≥ 16 GiB (rec. 32) | `NeedsUserAction` | `preflight.insufficient_disk` | `GetDiskFreeSpaceEx` |
   | Admin / elevation | UAC elevation obtainable | `Blocker` | `preflight.elevation_unavailable` | `CheckTokenMembership`(Administrators) |
   | **Enterprise policy (office-PC critical)** | WSL not policy-blocked; Store usable; CFA not blocking | `Blocker`/`NeedsUserAction` | `preflight.wsl_blocked_by_policy`, `preflight.store_disabled`, `preflight.controlled_folder_access` | Group Policy/MDM registry, Store state, Defender Controlled Folder Access |

   **Not gated:** Windows edition (WSL2 runs on Home via VirtualMachinePlatform). WSL app version (≥2.4.4) checked at wsl-enable after `--update`. Network is a non-blocking ping; real failures surface later (`common.network_failed`, `common.proxy_tls_error`).
   **Elevation nuance:** `preflight.elevation_unavailable` (`Blocker`) is for a non-admin account with no obtainable admin; a user clicking "No" on UAC mid-flow is `wsl.elevation_denied` (`NeedsUserAction`, retryable).

2. **WSL enable/update** — `wsl --install --no-distribution` + `wsl --update` via UAC elevation; verify WSL app ≥ 2.4.4. **Reboot-resume:** before a required reboot, write `Cowork.exe --resume` to the Windows **`RunOnce`** registry key so the wizard relaunches automatically (a non-developer will not relaunch manually); persist wizard state; on resume, validate state (`host.resume_state_corrupt`). Note the `wsl --update` Catch-22: build 19041's inbox `wsl.exe` may not support modern flags before update — handle via `wsl.update_unsupported_inbox` (guide to Store/MSI update).

3. **Provision distro** — create the **Cowork-dedicated distro** (decision #2), never disturbing an existing `Ubuntu`. Cover: for the `--import` path, rootfs fetch (`distro.rootfs_fetch_failed`) + SHA256 verify (`distro.checksum_mismatch`) + import (`distro.import_failed`); for the `--name` path, `distro.install_failed`; an existing `Cowork` distro → idempotent re-use or explicit re-provision (`distro.already_exists`); firstboot user creation (`distro.user_create_failed`); disk exhaustion (`distro.disk_full`). Then **inject the guest CLI**: the host copies the cross-compiled Linux `cowork` binary into the distro (via `\\wsl.localhost\Cowork\...` or piping through `wsl.exe`) before invoking `wsl.exe -d Cowork -- cowork ...` (`guest.cli_inject_failed`).

4. **Toolchain bootstrap** (`cowork` CLI, inside WSL) — `apt install` Linuxbrew prerequisites first (`build-essential`, `curl`, `git`, `file`, `procps` → `toolchain.prereq_apt_failed`); install Linuxbrew (`/home/linuxbrew/.linuxbrew`) + mise; wire shellrc (`brew shellenv`, `mise activate`). Locales: `locale-gen` ja_JP/ko_KR/en_US. ~~If codex selected, pin Node 24 LTS via mise.~~ **Dropped — codex is standalone; no Node is installed.** mise ships anyway, for the *user* to manage runtimes in a workspace without sudo.

5. **Agent install** (decision #9) — **all three** via their own native installer, run with unattended flags to avoid TTY hangs (`agent.installer_hang`); codex additionally needs `CODEX_NON_INTERACTIVE=1`, because its installer prompts by reading `/dev/tty` directly. Verify each binary post-install (`agent.binary_not_found`, `agent.integrity_check_failed`). ~~codex via `~/.default-npm-packages` on the pinned Node. Route creds to `~/.cowork/creds/`.~~ **Both reversed — creds stay at each agent's default path.**

6. **Auth** — ~~run each agent's native login.~~ **Reversed (`816317a`).** Cowork runs **no** login command and pre-flights no auth. A tab launches the agent bare; the agent prompts for its own sign-in. The host's only job is to notice a printed OAuth URL and open the real browser. Credentials land wherever the agent puts them, at its default path.

7. **Done** — drop the user into the embedded terminal at **`~/workspaces/default/`** (the single workspace seeded in v0.1; workspaces live under a `~/workspaces/` container — one per agent session — so v0.2 per-workspace isolation adds siblings with no migration).

## Uninstall (v0.1, minimal)

A **Remove Cowork** action does a clean teardown: `wsl --unregister Cowork` (drops the dedicated distro and all its data), then clears host-side state (the `RunOnce` key if pending, persisted wizard state, `~/.cowork` host config). Clean precisely because the install is isolated to a dedicated distro — the user's other distros are untouched. Failure → `distro.unregister_failed`.

## Architecture / crates

- **`src/`** — SvelteKit-static UI: wizard + embedded terminal. i18n via Paraglide-JS (ja+ko+en, en base).
- **`src-tauri/`** — Tauri v2 Rust, **Windows host driver** (OS-specific layer): preflight queries (incl. policy checks), `wsl.exe` driver, UAC elevation + `RunOnce` reboot-resume, ConPTY↔`wsl.exe` PTY bridge to xterm.js, structured progress events, **guest-CLI injection**. Builds `Cowork.exe`. Applies the 7 embedded-terminal tunings.
- **`cowork/`** — **host-agnostic guest CLI** (binary `cowork`) run inside WSL; performs bootstrap (apt prereqs → brew → mise → shellrc → locales → workspace → agent install), emits JSON-lines progress. ~~Owns `~/.default-npm-packages` and the `~/.cowork/creds` routing.~~ **Neither exists.** Unit-testable (`cargo test -p cowork`). Future `coworkd` daemon (v0.5) lands beside it; reused as-is on any future host.
- **`templates/`** — minimal default workspace only in v0.1.
- (`distro-build/` is **not** in v0.1 — deferred with Stage B.)

## Error model — LOCKED (post-review)

The `cowork` CLI and `src-tauri` backend **never emit localized strings** — only a structured envelope; the frontend maps it to localized text.

- **Envelope:** `{ code, kind, stage, context, cause? }`.
  - `code` — stable dotted `<domain>.<reason>`; also the i18n key root (`errors.<code>.title`/`.body`). No numeric codes.
  - `kind` — `Blocker` (unbypassable here) · `NeedsUserAction` (fix outside app, retry) · `Transient` (retry/backoff) · `Internal` (bug → report).
  - `stage` — `preflight | wsl-enable | provision | toolchain | agent-install | auth | done`.
  - `context` — camelCase key-values for ICU interpolation. `cause` — optional English diagnostic for logs/support, never shown localized, and **redacted** (strip usernames, home paths, tokens, URLs, OAuth artifacts) before any support export.
- **Single source of truth:** one checked-in `errors.json` (`code → {kind, stage, contextKeys}`); `build.rs` codegens the Rust enum; the TS frontend imports the same file. Parity across Rust ↔ TS ↔ 3 locales is guaranteed by construction.
- **Network rule:** transient connectivity failures collapse to `common.network_failed` (Transient, keyed by `stage`); endpoint-specific failures (changed/404 installer URL, TLS interception) keep their own code because remediation differs.

## Operational diagnostics (known fragile paths)

The external-installer + auth steps are the top v0.1 failure variables and are explicitly tracked as **fragile/external** in a (redacted) support log bundle: agent installer URLs (`claude.ai/install.sh`, `antigravity.google/cli/install.sh`, npm registry), TLS/proxy interception, the OAuth flow, and each agent's actual credential path. These are **re-validated whenever an agent self-updates** (URLs/paths can change without notice — see WP7 pre-gate). The bundle redacts usernames / home paths / tokens / URLs / OAuth artifacts per the `cause` rules.

## Out of scope for v0.1 (explicit)

bubblewrap isolation · snapshots/recovery · observability/budget · GitHub community backend · workspace-management UI · credential vault (creds stay at each agent's own default path inside the distro — the distro *is* the boundary) · template marketplace · custom baked rootfs (Stage B).

## Quality & conformance (applies to every WP)

Functional "Done when" is necessary but **not sufficient** — it guards against drift from the plan. A unit/WP is complete only when **all four** hold:

1. **Functional DoD** — the per-WP checks below.
2. **Automated conformance passes** (CI/lint/test — the *unit-seal gate*): host/guest separation (`cowork` has no Windows deps), no backend-emitted localized strings (codes only), `errors.json` single-source + 3-locale parity, envelope-only guest→host messaging. A unit failing these **cannot be sealed/committed**. (Full rule list in [`../AGENTS.md`](../AGENTS.md) → Quality & conformance.)
3. **Separate-context review sign-off** — architecture-conformance + code review by a *different* agent, with evidence; never self-approved. Risk-weighted: heaviest on WP4–WP8, light on WP0–WP2.
4. **Deviation recorded** — commit trailer for small departures; `docs/adr/NNNN-*.md` for architectural ones (with temporary/permanent + v0.2–v0.5 impact).

**Incremental integration:** WP4–WP8 each run their e2e slice on the Hyper-V checkpoint VM **as they finish** — integration is continuous, not deferred to WP10.

## Implementation work breakdown (with acceptance criteria)

Every WP lists what to build and **Done when** = the concrete, observable check that proves it is correctly implemented. "unit" = automated test; "e2e" = on the Hyper-V clean-room (WP10).

**Build order / dependencies:** WP0 → WP1+WP2 (parallel) → WP3 → WP4 → WP5 → WP6 → WP7 → WP8 → WP9 → WP10. WP8/WP9 (UI) can start against mocks once WP1/WP2 land.

### WP0 — Repo scaffold & docs
- Materialize `docs/plan.md` + `docs/v0.1.md` (English). Create the `cowork/` guest-CLI crate; wire SvelteKit-static + Tauri v2 (`src/`, `src-tauri/`), `package.json` scripts. Finalize `Cargo.toml` workspace dependencies for actual v0.1 use.
- **Done when:** `cargo build` compiles the workspace; `cargo metadata` lists members `src-tauri` + `cowork`; `npm run build` (pnpm) emits the static bundle; `docs/v0.1.md` contains the full error-code list; `cargo tauri dev` launches an empty shell window.

### WP1 — i18n foundation
- Paraglide-JS (`sourceLanguageTag: "en"`), catalogs `ja/ko/en` with ICU interpolation; Windows locale auto-detect → supported locale else en; language selector on the first install screen; missing-key fallback → en.
- **Done when:** unit — switching locale swaps all visible strings; an unknown key renders the en string (not the key). Manual — ja/ko Windows boots into ja/ko; selector overrides; `Intl.NumberFormat`/`DateTimeFormat` used (no hardcoded number/date formats, grep-verified).

### WP2 — Error model (single source)
- `errors.json` registry (`code → {kind, stage, contextKeys}`) holding all 53 codes; `build.rs` codegens the Rust enum; TS frontend imports the same file. Envelope `{code,kind,stage,context,cause?}` types both sides. `cause` redaction function.
- **Done when:** unit — a test asserts the set of Rust enum variants == `errors.json` keys == `errors.<code>.{title,body}` keys present in **all 3** locales (build fails on any drift); envelope round-trips dotted codes through serde; redaction test strips usernames, home paths, tokens, URLs, OAuth artifacts from a sample `cause`. Each `kind` has a defined UI treatment (referenced by WP9).

### WP3 — Preflight (host, `src-tauri`)
- All checks in the Setup-flow §1 table: Windows build, arch, virtualization capability + firmware-enabled (**cross-check `HypervisorPresent`**), hypervisor conflict, free disk ≥16 GiB on the WSL target volume, elevation, enterprise policy (WSL-blocked / Store-disabled / Controlled-Folder-Access). Each emits its `preflight.*` code with the correct `kind`.
- **Done when:** unit — mocked Windows queries drive each check across pass / fail / threshold-boundary, asserting the exact code+kind; the Hyper-V-owns-VT-x case returns *enabled* (no false-negative). e2e — on a VT-x-disabled VM the wizard shows the localized "enable virtualization" blocker and does **not** proceed; a sub-16-GiB volume blocks with `preflight.insufficient_disk`.

### WP4 — WSL enable/update + reboot-resume (host)
- `wsl --install --no-distribution` + `wsl --update` via UAC; verify WSL app ≥2.4.4; inbox-wsl Catch-22 → `wsl.update_unsupported_inbox`. Reboot-resume: write `Cowork.exe --resume` to **`RunOnce`**, persist wizard state, validate on resume (`host.resume_state_corrupt`).
- **Done when:** unit — state persist/restore round-trips; corrupt state → `host.resume_state_corrupt`; UAC-decline path → `wsl.elevation_denied` (retryable, not Blocker). e2e — from a clean checkpoint (WSL off): wizard enables WSL, reboots, **auto-relaunches via RunOnce**, resumes at the correct step without user action; `RunOnce` key is consumed (not left to loop).

### WP5 — Distro provisioning + guest-CLI injection (host)
- Provision the **Cowork-dedicated distro** (decision #2: `--name Cowork`, or `--import Cowork` of a verified vanilla Ubuntu rootfs); **must not touch an existing `Ubuntu`**. Handle rootfs fetch/checksum/import (or install) failures, `distro.already_exists` (idempotent re-use), `user_create_failed`, `disk_full`. Cross-compile `cowork` (Linux x64+arm64), inject (`\\wsl.localhost\Cowork\…` copy or pipe), then `wsl.exe -d Cowork -- cowork …`; parse JSON-lines on the host. Also implement **Remove Cowork** (distro lifecycle): `wsl --unregister Cowork` + host-state cleanup (`distro.unregister_failed`).
- **Done when:** unit — host JSON-lines parser handles a sample stream incl. `protocol.parse_error` / `protocol.version_mismatch`. e2e — a `Cowork` distro is created **while a pre-existing `Ubuntu` distro is left untouched** (`wsl -l -v` confirms both; Ubuntu unchanged); `wsl -d Cowork -- cowork --version` runs with matching arch; re-running provisioning re-uses the existing `Cowork` distro (no duplicate); **Remove Cowork** unregisters only the `Cowork` distro (other distros intact) and clears host state.

### WP6 — Toolchain bootstrap (guest, `cowork` CLI)
- `apt install build-essential curl git file procps`; Linuxbrew + shellenv; mise + activate; `locale-gen` ja/ko/en; create the default workspace `~/workspaces/default/`. Emit JSON-lines progress per step. ~~Node 24 LTS exact-pin via mise if codex selected.~~ **Dropped (`f6159b2`) — codex is standalone.**
- **Done when:** unit (`cargo test -p cowork`) — command construction + progress emission per step; each failure path yields the right `toolchain.*` code; bootstrap is idempotent on re-run. e2e — `brew --version`, `mise --version` present; `node --version` is 24.x iff codex selected; the three locales exist (`locale -a`); `~/workspaces/default/` exists.

### WP7 — Agent install (guest)
- **Pre-gate (before building this WP):** re-spike Antigravity (install URL, binary `agy`, token path, OAuth flow) and re-confirm claude/codex installer URLs — these are externally controlled and may have drifted since the 2026-06 spike; abort/adjust if changed.
- All three via native installers with **unattended flags** (guard `agent.installer_hang` via timeout); codex needs `CODEX_NON_INTERACTIVE=1`. Post-install binary verification. ~~Creds routed to `~/.cowork/creds/` via env override or symlink.~~ **Reversed (`eab7522`) — creds stay at each agent's default path.**
- **Done when:** e2e — for each selected agent, `claude`/`agy`/`codex --version` works on PATH; a missing post-install binary raises `agent.binary_not_found`. ~~The npm/default-npm-packages unit tests, and the check that tokens land under `~/.cowork/creds/`.~~ **Both obsolete** — there is no npm path, and creds are the agent's own business.

### WP8 — Embedded terminal + PTY bridge + auth (host + UI)
- ConPTY ↔ `wsl.exe` PTY bridge to xterm.js + the 7 tunings. Auth: run each agent's native login; OAuth handoff to host browser via **WSLInterop** (`$BROWSER`/`xdg-open`); fallback = print URL / device-code (`auth.browser_launch_failed` / `auth.oauth_callback_failed` / `auth.login_timeout`).
- **Done when:** e2e — terminal is interactive (keystroke echo, resize reflow, 256-color); running agent login **opens the host browser from inside WSL** and completes OAuth; with Interop disabled, the manual-URL fallback completes login; timeout path emits `auth.login_timeout`.

### WP9 — Wizard UI flow (UI)
- **Onboarding screens before the 7 technical steps:** language selector (first screen) + **agent selection** (which of claude/codex/antigravity to install, 1+) — the selection feeds step 5. Then the 7-step wizard; per-step progress from JSON-lines; per-`kind` UI treatment (Blocker = stop+explain; NeedsUserAction = instructions+Retry; Transient = auto-retry/backoff+manual Retry; Internal = report+log path).
- **Done when:** unit/component — each `kind` renders its prescribed affordance; a `Transient` error auto-retries then offers manual retry; an `Internal` error surfaces the (redacted) `cause` + log path. e2e — the happy path walks all 7 steps and lands the user in the terminal.

### WP10 — End-to-end integration & clean-room verification (authoritative)
- **Clean-room = Hyper-V VM (Win11 guest, nested virt on: `Set-VMProcessor -ExposeVirtualizationExtensions $true`) + checkpoints**; revert = instant "WSL never enabled". Windows Sandbox is **not usable** (no nested virtualization).
- **Two gates (cost-split):**
  - **PR smoke gate (fast, automatable):** mocked/headless host-driver + guest bootstrap on a WSL-already-enabled box, single agent (codex, OAuth stubbed), no reboot. Runs per PR to catch regressions cheaply.
  - **Nightly / release full gate (heavy, partly manual):** from a reverted clean checkpoint, the full from-nothing + reboot + **real OAuth** path. OAuth is interactive (browser) and **not fully CI-automatable** → manual or semi-automated, run nightly/pre-release, not per-PR.
- **Done when (v0.1 definition of done — full gate):** from a reverted clean checkpoint, for **each** agent selection (claude / codex / antigravity, and a multi-select): WSL installed → **`Cowork` distro provisioned with any existing `Ubuntu` left untouched** → `brew`+`mise`(+`node` 24.x if codex) present → selected agent installed and `--version` works → real native login completes (incl. WSLInterop browser handoff) → embedded terminal interactive in the working dir. Re-running is idempotent (no duplicate `Cowork` distro). Every induced failure (VT-x off, <16 GiB, UAC decline, network drop, existing-`Cowork` conflict) shows the correct localized message and recovery affordance.
