# WP6 — Full gate runbook (v0.2 definition of done)

The **authoritative** v0.2 acceptance gate. Like the v0.1 gate it runs **manually /
semi-automated** on a real Windows clean-room; everything cheaply automatable already runs
per-push in `ci.yml` (frontend, guest/host unit tests, conformance, docker `guest-e2e`,
real-WSL2 `wsl-integration`, and the windows-latest `host` job that compiles `src-tauri`).
This gate covers what those cannot: the real ConPTY multi-session lifecycle, real agent
resume across app restarts, real OAuth-state-driven auth gates, and the v0.1 → v0.2
**upgrade** path.

**Clean-room mechanics (VM creation, checkpoints, baseline verification, i18n passes) are
identical to v0.1 — follow [`wp10-full-gate.md`](./wp10-full-gate.md) §1–§3 and reuse
`scripts/wp10/fullgate.ps1`.** This document only defines what v0.2 adds or changes.

Scope sealed on `main` as of `2f9a6d1`: WP0–WP5 + boot-time guest re-injection
(multi-session PTY `7d08d9b`, session tabs + claude resume `9184f48`, codex/agy
continuity `479ee4d`, auth self-driving `dc0b42f`, presets `37b1bf8`, upgrade healing
`2f9a6d1`).

## 2. Entry paths

Run **both**. Path A is the new release-blocking coverage; Path B re-validates the wizard
with the v0.2 deltas.

- **Path A — upgrade healing:** revert to `v01-setup-complete` → install/replace with the
  v0.2 `Cowork.exe` → launch. Continue at §3.
- **Path B — fresh install:** revert to `ready-to-install` → install v0.2 → launch.
  Continue at §4 (wizard), then everything from §5 onward.

## 3. Path A — upgrade healing (boot-time guest re-injection)

| Step | Action | Expected (PASS) |
|------|--------|-----------------|
| 3.1 | Before first launch, fingerprint the stale guest: `wsl -d Cowork -- sha256sum /usr/local/bin/cowork` | hash recorded (this is the v0.1 guest) |
| 3.2 | Launch the v0.2 app | boot gate skips the wizard (marker honored) and lands in the **shell** (sidebar + default workspace), not the wizard |
| 3.3 | Re-run the sha256 from 3.1 after the shell has loaded | **hash changed** — `guest_sync` re-injected; binary is executable (`wsl -d Cowork -- /usr/local/bin/cowork --help` lists `session-uuid`, `agent-theme`, `auth-status`) |
| 3.4 | Create a workspace with the **PDF translate** preset | succeeds (proves the new guest's `--preset` arg end-to-end); `AGENTS.md` + `CLAUDE.md` appear per §5.3 |
| 3.5 | Second app launch | boot is NOT slower than the first (sync compares bytes and skips injection — no repeated copy) |
| 3.6 | v0.1 data intact | the v0.1 `default` workspace and its `files/` content survived; metadata (`%LOCALAPPDATA%\Cowork\workspaces.json`) still lists it |

## 6. Sessions — multi-PTY, tabs, glyphs

| Step | Action | Expected (PASS) |
|------|--------|-----------------|
| 6.1 | Visit workspace A (claude autoruns), start a long agent task; switch to workspace B; wait; switch back | A's session **kept running** (output progressed while hidden); no respawn, scrollback intact; TUI not corrupted after the switch (no 0×0 resize artifacts) |
| 6.2 | Tab strip: ⊕ | new tab with the workspace default agent, title `Claude 2`-style, becomes active |
| 6.3 | ▾ picker → another agent | session spawns with that agent AND the workspace default agent updates (next ⊕ uses it) |
| 6.4 | Status glyphs | cold (restored, not yet activated) → idle after spawn → **working pulse** while the agent streams → idle ~2 s after output stops → **done** when the process exits (type `exit` in a shell-ish session or `/quit` the agent) |
| 6.5 | Second concurrent session in the same workspace | one-shot advisory bar appears (shared-files warning); dismissible; does not reappear for the third |
| 6.6 | Close a tab (×) | PTY killed, metadata updated; closing the **last** tab immediately opens a fresh default-agent tab |
| 6.7 | Close the app window with 2+ live sessions across workspaces | all `wsl.exe` ConPTY children die (check Task Manager / `Get-Process wsl` before/after) — the `WindowEvent::Destroyed → kill_all` backstop |

## 7. Session continuity (the v0.2 core promise)

For each agent — claude, codex, antigravity — in its own workspace:

| Step | Action | Expected (PASS) |
|------|--------|-----------------|
| 7.1 | New session, send a first message ("remember the word *kumquat-7*"), wait for the reply and ~5 s of quiet | for codex/agy: `workspaces.json` gains the session's `agentSessionUuid` (lazy capture after idle); claude's was set at creation |
| 7.2 | Quit and relaunch the app | tab strip rebuilt from metadata; tabs show **cold** glyphs; nothing spawns until activated (lazy restore) |
| 7.3 | Activate the tab | spawn uses the resume form (`claude --resume`, `codex resume`, `agy --conversation`); asking "what word did I ask you to remember?" returns *kumquat-7* |
| 7.4 | claude edge: create a session, type nothing, restart, activate | claude prints its own "no conversation found"-style error in the terminal (accepted); opening a new tab works |
| 7.5 | Two codex sessions in the SAME workspace, messages in both | each tab resumes its own conversation after restart (cwd+time-keyed capture attributed correctly) |

## 10. Failure induction (delta)

| Case | Induce | Expected (PASS) |
|------|--------|-----------------|
| 10.1 | Corrupt the installed guest: `wsl -d Cowork -u root -- bash -c 'echo broken > /usr/local/bin/cowork'`, relaunch the app | shell boots; guest re-injected (sha matches shipped); workspace create works |
| 10.2 | Break the probe path only (e.g. `chmod -x` the guest), launch, then watch first session mount | boot proceeds (sync/probe failures are best-effort); after the heal in 10.1's mechanism or a relaunch, features recover; errors that do surface are localized envelopes, not raw panics |
| 10.3 | Kill `wsl.exe` for a running session from Task Manager | tab flips to **done**; app stays responsive; new sessions spawn fine |
